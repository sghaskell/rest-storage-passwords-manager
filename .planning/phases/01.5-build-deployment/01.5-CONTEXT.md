## Phase Boundary

Production build hardening, Docker verification testing with automated CRUD tests via Playwright, packaging verification, and documentation update when we're ready to bump version. Goal is ensuring the app builds correctly, passes AppInspect cloud validation, and all core CRUD functionality works per gitflow.

<decisions>
## Implementation Decisions

### Production Build Configuration
- **D-01:** Minification + Terser for production build (R2 requirement locked)
- **D-02:** Sourcemap behavior: `devtool` = `"source-map"` in development mode, disabled in production mode via separate config or conditional logic
- **D-03:** Webpack externals: React and ReactDOM externalized, splunkjs/mvc stays bundled (not external) - follows existing pattern where only ready! is external currently

### Testing & Verification Approach
- **D-04:** Playwright for Docker host testing only (no CI/CD automation needed)
- **D-05:** Automated CRUD tests targeting: credential listing, create, update, delete operations against running Splunk instance via REST API endpoints
- **D-06:** Test coverage focuses on happy paths + critical error scenarios (auth failures, 404s, 400s) - not exhaustive edge cases

### Deployment & Packaging
- **D-07:** Keep legacy `password-crud.js` bundled until v3.0 fully verified in target environment
- **D-08:** Version bumping and documentation updates follow standard gitflow when ready to release (docs created as part of version bump, not pre-built)

### Code Quality Targets
- **D-09:** Console.log/console.error removed from production builds (Terser drop_console)
- **D-10:** Bundle size target: under 2MB uncompressed for modern Splunk browser environments

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `API.getAllCredentials()` — Main read operation to list credentials from Splunk REST (getApps(), getUsers(), getRoles() for reference data)
- `API.createCredential(username, password, realm)` — POST credential creation with CSRF handling built-in
- `API.updateCredential(name, readRoles, writeRoles, owner, sharing?, app?)` — PUT credential updates all fields including ACLs
- `API.deleteCredential(name, sharing?, app?)` — DELETE credential and verify removal works across different sharing scopes
- **Test targets:** These functions can be called directly by tests or tested indirectly through the Playwright dashboard interactions since they're exposed on `window.CredentialManager.API` if needed

### Established Patterns
- REST API calls use splunkd/ URL format with `{owner}/{app}` namespace logic
- CSRF token required for all mutating requests (POST/PUT/DELETE) — tests must capture form key first
- Splunk session authentication via cookie or basic auth (tests can leverage admin credentials from `.env`)

### Integration Points
- Dashboard XML (`credential_management.xml`) has testable mount point with error display UI
- Install script handles local Docker testing needs out of the box
</code_context>
