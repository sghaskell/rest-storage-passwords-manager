# Phase 1.3 Context — Legacy Port Decisions

**Purpose:** Capture exact behavioral patterns from `password-crud.js` that must be replicated in React. This is NOT a gap list (see GAP-AUDIT.md) but rather the **source-of-truth port decisions** derived from reading all 1171 lines of legacy code.

---

## API Layer Decisions (api.js rewrites needed)

### ACL Operations — POST, not PUT; `perms.read`, `perms.write` format
All ACL updates in legacy use `splunkdPOST(buildAclPath(row), { 'perms.read': ..., 'perms.write': ..., sharing, owner })`. React must:
- Use **POST** (not PUT) for all ACL requests to `/configs/conf-passwords/...acl`
- Body fields: `'perms.read'`, `'perms.write'` (with dots), `sharing`, `owner`
- The existing `buildAclPath()` in api.js line 124 is correct (uses `credential:` prefix)

### Error Extraction
Legacy extracts XML `<msg>` from error responses: `text.match(/<msg[^>]*>([^<]+)<\/msg>/)?.[1]`. React currently dumps raw XML. The `apiRequest`/`splunkdRequest` functions must extract this text before throwing.

### CSRF + GET Requests
Legacy `splunkdFetch` always sends CSRF token for non-GET methods, and appends `output_mode=json&count=0` in the URL path itself (not in headers). React's `apiRequest` incorrectly adds `output_mode=json` to ALL requests including non-GET — this may cause issues with Splunk endpoints that don't expect query params on POST/DELETE.

## CRUD Flow Patterns (MUST match legacy exactly)

### Create: Two Sequential Requests
```
1. POST /servicesNS/{owner}/{app}/storage/passwords → { name: username, password, realm }
2. POST /servicesNS/{owner}/{app}/configs/conf-passwords/credential:{realm}:{username}:acl
   → { 'perms.read': read, 'perms.write': write, sharing: ... }
```
If `sharing === 'user'`, insert an extra step between 1 and 2: POST ACL with `sharing: 'app'` first, then POST with `sharing: 'user'`.

**React currently sends `owner` and `app` in the POST body** — legacy only uses them in the URL path. The body is `{ name: username, password, realm }` only.

### Update: Four-Step Sequence
```
1. POST buildAclPath(row) → { 'perms.read': read, 'perms.write': write, sharing: 'app', owner }  // bump to app first
2. (if password changed) POST /servicesNS/nobody/{row.app}/storage/passwords/{row.stanza} → { password }
3. (if app changed) POST /servicesNS/nobody/{row.app}/configs/conf-passwords/credential:{row.stanza}/move → { app: newApp, user: 'nobody' }
4. POST /servicesNS/nobody/{newApp}/configs/conf-passwords/credential:{row.stanza}/acl → { 'perms.read': read, 'perms.write': write, sharing, owner }  // final ACL with requested sharing
```

Key detail: Step 2 uses `servicesNS/nobody/` (not `servicesNS/{owner}/`). Step 1 always sets `sharing: 'app'` first. The move in step 3 is a SEPARATE endpoint, not part of the credential POST body.

### Delete: ACL Bump Before DELETE
```
1. POST buildAclPath(row) → { 'perms.read': row.acl_read, 'perms.write': row.acl_write, sharing: (row.acl_sharing === 'user') ? 'app' : row.acl_sharing, owner: row.owner }
2. DELETE /servicesNS/{owner}/{app}/storage/passwords/{stanza}
```

For bulk delete, use `Promise.allSettled` so partial failures are reported per-row instead of aborting on first failure.

### Password Reveal: User Sharing Bump Pattern
When credential `sharing === 'user'`, legacy temporarily bumps sharing to `'app'`, fetches the password, then reverts to `'user'`. React's `PasswordRevealModal` has no API call — needs full implementation with this bump pattern.

## Dropdown Lookups (fetchApps, fetchRoles, fetchUsers)

All three must be added to api.js:
- `getApps()`: GET `/servicesNS/-/-/apps/local?output_mode=json&count=0&search=disabled%3D0` → `{ label: content.label || name, value: name }`
- `getRoles()`: GET `/servicesNS/-/-/authorization/roles?...` → prepend `{ label: '* (all roles)', value: '*' }` first option
- `getUsers()`: GET `/servicesNS/-/-/authentication/users?...` → `{ label: name, value: name }`

## Form Patterns (CredentialForm.jsx)

### Multi-Select for Roles
Legacy has custom multi-select behavior:
- `* (all)` is mutually exclusive with individual roles — selecting * deselects all others, and vice versa
- "Select All" button selects every NAMED role but NOT *
- "Reset" button restores default selections
- Counter shows `${N} selected`

### App Scope Warning
When selected app equals `getCurrentApp()`, show a conditional warning hint: "Credentials are stored in this app's local directory and will be lost if it is uninstalled."

### Realm Field Disabled on Edit
Legacy disables the realm input during edit: `realmInput.disabled = true`. Realm cannot be changed via REST API after creation.

### Validation Rules (from legacy handleCreateCredential/handleUpdateCredential)
```
Create:
- Username required
- Password required
- password === confirmPassword required
- read roles non-empty
- write roles non-empty

Update:
- If password field is filled, must match confirmPassword
- read roles non-empty
- write roles non-empty
```

## Table Behaviors (CredentialTable.jsx)

### Client-Side Filtering
Legacy filters by username, realm, or app using case-insensitive substring match. Filter input value persists across re-renders. Reset to page 1 on filter change.

### Pagination
PAGE_SIZE = 10. Paginator hidden when total rows ≤ PAGE_SIZE. Clamp current page if last page is deleted.

### Selection
Checkbox column with select-all/indeterminate state in header. Delete button enabled only when ≥1 row selected. Bulk delete shows confirmation dialog with all names listed.

## CSV Import Pipeline (if porting to Phase 1.3, otherwise defers to 1.4)

- RFC 4180 parser handles quoted fields with escaped quotes (`""`)
- Lines starting with `#` are comments — skip them
- Required columns: `username`, `password`
- Default values for optional columns: app → current app, owner → current user, sharing → `'app'`, read/write → `'admin,power'`
- Owner column rejects `*` (must be real user, falls back to current user)
- File size guard: 512KB max
- Preview modal shows parsed rows with masked passwords
- Batch create uses `Promise.allSettled`; results modal shows per-row status
- 409 → "already exists" (not failure)

## Current App Detection
Legacy reads app context from URL: `window.location.pathname.match(/\/app\/([^/]+)/)`
React needs equivalent or Splunk utility call to determine current app for defaults.
