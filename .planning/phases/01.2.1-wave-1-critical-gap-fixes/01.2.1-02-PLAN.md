```js
async function fetchClearPassword(restUri) {
    const res  = await splunkdGET(`${restUri}?output_mode=json`);
    const json = await res.json();
    return json.entry?.[0]?.content?.clear_password ?? null;
```

    ```js
    async function getCredentialPassword(name, realm) {
        const encodedName = encodeURIComponent(name);
        const encodedRealm = encodeURIComponent(realm);
        // Use storage/passwords endpoint — GET by path
        const data = await apiRequest(`/${encodedRealm}:${encodedName}`);
        return (data.entry && data.entry[0] && data.entry[0].content)
            ? data.entry[0].content.clear_password || null
            : null;
    }
    ```

    Add `moveCredential(name, realm, newApp)` function matching legacy pattern (password-crud.js line 533-539). The credential is moved from its current app to the target app via POST to the `/move` endpoint.

    ```js
    async function moveCredential(name, realm, newApp) {
        const encodedName = encodeURIComponent(name);
        const encodedRealm = encodeURIComponent(realm);
        const stanzaKey = `${encodedRealm}:${encodedName}:`;
        const credId = encodeURIComponent(`credential:${stanzaKey}`);
        
        await splunkdRequest(
            `/servicesNS/nobody/${encodeURIComponent(newApp)}/configs/conf-passwords/${credId}/move`,
            {
                method: 'POST',
                body: {
                    app: newApp,
                    user: 'nobody',
                },
            }
        );
    }
    ```

    Export in module.exports. Note: The current `updateCredential` function doesn't call moveCredential — it passes `app` in the POST body to storage/passwords which Splunk ignores for credential updates (per GAP-C07). The callers in bundle.jsx need to be aware of this, but that wiring is outside Wave 1 scope — the function just needs to exist and work correctly.

    NOTE on stanza encoding: Use `credential:${realm}:${name}:` with each component URL-encoded separately, then the full string double-encoded for the path segment. This matches legacy exactly: `encodeURIComponent(\`credential:${row.stanza}\`)`.
