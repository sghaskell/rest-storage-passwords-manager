## Phase 01.4 Plan 01: Form UI Upgrades - Dropdowns, Sharing, Password Confirmation

### Objective

Replace free-text fields in CredentialForm with validated dropdowns and multi-select controls, add password confirmation on create/update, and add a sharing selector. This form serves as the user-facing contract for credential metadata. All downstream API changes (plan 02) consume this output.

**Purpose:** Prevent typos in app/realm/owner names that silently fail against Splunk REST API, enable full RBAC control via validated role selection and sharing scope, and eliminate password-entry errors with confirmation matching.

### Context

Phase 01.3 (VERIFIED 10/10) wired these props into CredentialForm:
- `availableApps` - array of `{name}` objects from `/apps/local`
- `currentUserIdentity` - string username from `/authentication/current-context`
- `availableRoles` - array of role name strings from `/authorization/roles`
- `defaultReadRoles` / `defaultWriteRoles` - "admin, power" constants

CredentialForm currently ignores these props entirely (form only destructures `{ credential, onSave, onCancel }`). Phase 01.2 left app/owner/free-text roles in the form - this plan replaces them with validated controls per legacy patterns.

### Key Legacy References (password-crud.js)

```javascript
// App dropdown: L153-156 fetches /apps/local, builds select options
// Owner: L170-173 defaults to current user from /authentication/current-context
// Sharing options: { global: 'All Apps', app: 'App', user: 'Specific User' }
// Two-step ACL on create for sharing='user': L478-481 (plan 02 scope)
// Password confirm validation on create: L490 - password !== confirmPassword -> error modal
// Password confirm validation on update: L514 - same check when password provided
```

#### Task 2: Multi-select roles replacing textarea + password confirmation (V20, V01, V02)

**Files:** `appserver/static/react/components/CredentialForm.jsx`

**Action:**

1. Replace readRoles/writeRoles textareas with multi-select checkbox grids:

3. Add "Confirm Password" field:
   - New state: `const [confirmPassword, setConfirmPassword] = useState('')`
   - New state: `const [errors, setErrors] = useState({})` for inline validation errors (string map by field name)
   - On create mode: render "Confirm Password" required input immediately after password field
   - On edit mode with `isChangingPassword`: show "Confirm Password" conditionally
   - Clear confirmPassword state when toggling off

4. Add pre-submit validation in handleSubmit:
   ```javascript
   if (!credential || isChangingPassword) {
       if (password !== confirmPassword) {
           setErrors({ passwordMismatch: 'Passwords do not match' });
           return;
       }
   }
   ```
   - Render error message in red below confirm field when `errors.passwordMismatch` is set

5. Update handleSubmit payload serialization to use arrays directly:
   ```javascript

   ```

**Verify:** Build success. DOM inspection: checkbox inputs for roles present; Confirm Password field visible on both create and edit+changePassword modes. Inline error message renders on mismatch before submit.
