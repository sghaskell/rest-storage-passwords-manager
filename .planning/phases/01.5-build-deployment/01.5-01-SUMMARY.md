## Decisions Made
- **Webpack function export**: Used `module.exports = (env, argv) => { ... }` to access `argv.mode` for conditional devtool. This is the standard webpack pattern and avoids relying on environment variables.
- **TerserPlugin explicit import**: While webpack 5 includes TerserPlugin by default in production mode, importing it explicitly allows overriding defaults with `drop_console: true`.
- **CamelCase externals**: Used `'React'` (not `'react'`) and `'ReactDOM'` for the global names — this matches Splunk's exposed global naming convention.
