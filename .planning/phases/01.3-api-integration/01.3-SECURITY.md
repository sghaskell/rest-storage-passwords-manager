# Phase 01.3 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Component→apiRequest | All API calls pass through splunkd/__raw proxy with CSRF | Credential data (usernames, passwords, ACLs) |
| getApps/getUsers/getRoles → Splunk endpoints | New GET requests to /apps/local, /authentication/*, /authorization/roles | User identity, app metadata, role names |
| bundle.jsx alert() messages | Parsed error strings shown to user via browser native alert | Error text from Splunk XML responses |
| CredentialForm props → child component | Data passed via React.createElement (not innerHTML) | App lists, user identity, role lists |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-01.3-01 | Information Disclosure | `getUsers()` fetches current-context | Accept | Returns only current authenticated user identity; no enumeration possible | **closed** |
| T-01.3-02 | Elevation of Privilege | `getRoles()` lists all roles | Accept | Read-only endpoint; role names are public metadata in Splunk | **closed** |
| T-01.3-03 | Tampering (XSS) | `parseError` extracts text from XML | Mitigate | Regex extraction (api.js:16) + HTML entity stripping (api.js:24-29) + React.createElement auto-escapes; no innerHTML/dangerouslySetInnerHTML in codebase | **closed** |
| T-01.3-04 | Information Disclosure | `getErrorMessage()` may leak internal Splunk paths | Mitigate | parseError extracts only `<msg>` content (api.js:16); status code and endpoint details never shown to user | **closed** |
| T-01.3-05 | Tampering (XSS) | Error text injected into alert() or React.createElement content | Accept | alert() is native browser function (no HTML rendering); React.createElement auto-escapes all text content; no dangerouslySetInnerHTML anywhere in codebase | **closed** |

*Status: open · closed*
*Disposition: mitigate (implementation required) · accept (documented risk) · transfer (third-party)*

### Audit Evidence

**T-01.3-01**: Verified `getUsers()` at api.js:355 only fetches `/authentication/current-context` — returns single user identity, no enumeration vector.

**T-01.3-02**: Verified `getRoles()` at api.js:376 fetches from read-only `/authorization/roles` endpoint — role names are public Splunk metadata.

**T-01.3-03**: Verified parseError at api.js:14 uses regex extraction (`/<msg[^>]*>([\s\S]*?)<\/msg>/i`) with HTML entity stripping (api.js:24-29). Confirmed no innerHTML or dangerouslySetInnerHTML usage across entire React codebase via grep.

**T-01.3-04**: Verified getErrorMessage at bundle.jsx:47 delegates to `API.parseError` which only extracts `<msg>` content — status codes, HTTP methods, and endpoint paths never shown to user.

**T-01.3-05**: Verified all error messages use native `alert()` (bundle.jsx:120, 135, 149) or React.createElement text nodes — neither renders HTML. Confirmed zero instances of dangerouslySetInnerHTML in codebase.
