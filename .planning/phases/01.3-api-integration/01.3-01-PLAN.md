# Phase 01.3 Plan 01: API Error Handling and Data Fetching Functions

Implement API-level helpers for better error handling and reference data source fetching that downstream components (Plan 02) will consume to produce validated dropdowns and readable error messages.

**Purpose:** Fill GAP-AUDIT items E03 (XML error parsing), E01 (409 conflict detection), V18/V19/V20 (missing dropdown data sources). Plan 02 depends on these exports being available.

**Output:** Enhanced api.js with `parseError`, `parseCreateError`, `getApps`, `getUsers`, `getRoles`, `DEFAULT_READ_ROLES`, `DEFAULT_WRITE_ROLES`

- `/servicesNS/-/-/apps/local?output_mode=json` → list of apps for App dropdown

- `/servicesNS/-/-/authorization/roles?output_mode=json` → all available roles

<task type="auto">
  <name>Task 2: Add getApps, getUsers, getRoles reference data fetching functions</name>
  <files>appserver/static/react/api.js</files>
  <action>
    Implement three new async functions using splunkdRequest pattern. Each gracefully catches errors and returns empty array/object instead of throwing — dropdowns degrade gracefully if APIs fail.

    ```javascript
    async function getApps() {
      try {
        const data = await splunkdRequest('/servicesNS/-/-/apps/local', { method: 'GET' });
        return (data.entry || []).map(e => ({ name: e.name }));
      } catch (err) {
        console.warn('Failed to fetch apps:', err.message);
        return [];
      }
    }

    async function getUsers() {
      try {
        const data = await splunkdRequest('/servicesNS/nobody/system/authentication/current-context', { method: 'GET' });
        const entry = (data.entry || [])[0];
        if (!entry) return { username: 'nobody' };
        return {
          username: entry.content?.name || entry.name || 'nobody',
          fullName: entry.content?.fullName || '',
          email: entry.content?.email || '',
        };
      } catch (err) {
        console.warn('Failed to fetch current user:', err.message);
        return { username: 'nobody' };
      }
    }

    async function getRoles() {
      try {
        const data = await splunkdRequest('/servicesNS/-/-/authorization/roles', { method: 'GET' });
        return (data.entry || []).map(e => e.name);
      } catch (err) {
        console.warn('Failed to fetch roles:', err.message);
        return [];
      }
    }
    ```
