## Legacy Reference Patterns (from password-crud.js)

### ACL Path Construction (line 456-459)
```js
// Stanza includes trailing colon: e.g., "prod:api-user:"
function buildAclPath(row) {
    const credId = encodeURIComponent(`credential:${row.stanza}`);
    return `/servicesNS/${encodeURIComponent(row.owner)}/${encodeURIComponent(row.app)}/configs/conf-passwords/${credId}/acl`;
}
```

```js
await splunkdPOST(`/servicesNS/${encodeURIComponent(owner)}/${encodeURIComponent(app)}/storage/passwords`, {

});
```

## Current React Bugs (from api.js)

### Bug C01/C08 — ACL path in updateCredential (line 205):
```js
// WRONG: missing "credential:" prefix, uses realm%3Aname format
const aclPath = `/servicesNS/nobody/${app || 'search'}/configs/conf-passwords/${encodedRealm}%3A${encodedName}/acl`;

// SHOULD be: credential:${stanzaKey} (URL-encoded)
```

### Bug C06 — createCredential name field (line 154):
```js
// WRONG: double-namespaces realm into both name AND realm field
name: realm ? `${realm}:${username}` : username,
realm: realm || '',

// SHOULD be: name = username only (Splunk combines them internally)
name: username,
realm: realm || '',
```

### Bug U04 — CredentialTable key (CredentialTable.jsx line 252):
```js
// WRONG: key based on username alone — collides for same user in different realms
{ key: cred.name || cred.id }

// SHOULD be: key based on stanzaKey — unique per credential
{ key: cred.stanzaKey }
```
