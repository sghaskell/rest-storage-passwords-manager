## Phase 01.4 Plan 01: Form UI Upgrades - Splunk React UI Native Components

### Objective

Replace CredentialForm's hand-rolled `React.createElement` + inline styles with native `@splunk/react-ui` components already installed in the project. Convert text inputs to Select, Multiselect, and properly-labeled Text fields. Add password confirmation and sharing selector.

**Purpose:** Use Splunk UI primitives (already in node_modules) to get accessible, themed form controls for free. Eliminates ~200 lines of inline style objects and React.createElement boilerplate. Prevents typos in app/realm/owner names, enables full RBAC via validated role selection, and eliminates password-entry errors.

### Splunk UI Components Available (Already Installed)

```
@splunk/react-ui version 5.9.1 (in devDependencies)
Components used:
  - ControlGroup: Wraps every field with label, help text, error support, accessibility
  - Select: Single-choice dropdown (app field, sharing field)
  - Multiselect: Multi-select with chips and filtering (read/write roles)
  - Text: Text input + password input (username, realm, owner, password fields)
  - Button: Cancel/Submit actions with proper appearance
```

**Docs reference:** `node_modules/@splunk/react-ui/docs-llm/`
- "All data entry components should be wrapped in a ControlGroup to provide a label, error states, and help or error text" (Select.md, Multiselect.md, Text.md)

### Context

Phase 01.3 wired these props into CredentialForm via bundle.jsx:
- `availableApps` - array of `{name}` objects from `/apps/local`
- `currentUserIdentity` - string username from `/authentication/current-context`
- `availableRoles` - array of role name strings from `/authorization/roles`
- `defaultReadRoles` / `defaultWriteRoles` - "admin, power" constants

CredentialForm currently ignores these props entirely (only destructures `{ credential, onSave, onCancel }`). Current implementation uses raw `React.createElement('input')`, `React.createElement('textarea')`, and inline style objects. Webpack babel-loader with `@babel/preset-react` is configured in webpack.config.js. Phase 01.2 used hand-rolled elements for simplicity; this plan upgrades to Splunk primitives.

#### Task 1: Convert to Splunk UI Components - ControlGroup, Select, Text, Button (V18, V19, C04)

**Files:** `appserver/static/react/components/CredentialForm.jsx`

**Action:** Comprehensive rewrite of CredentialForm to use native Splunk React UI components. Replace ALL `React.createElement` calls with JSX and Splunk primitives.

1. **Import Splunk components (replacing manual `require('react')` pattern):**
   ```javascript
   const React = require('react');
   const ControlGroup = require('@splunk/react-ui/ControlGroup').default;
   const Select = require('@splunk/react-ui/Select').default;
   const Multiselect = require('@splunk/react-ui/Multiselect').default;
   const Text = require('@splunk/react-ui/Text').default;
   const Button = require('@splunk/react-ui/Button').default;
   ```
   Note: Using `require().default` pattern to match existing CommonJS module style in the codebase (webpack.config.js babel-loader handles JSX transformation).

2. **Update component signature for new props:**
   ```javascript
   function CredentialForm({ credential, onSave, onCancel, availableApps = [], currentUserIdentity = 'nobody', availableRoles = [], defaultReadRoles = '', defaultWriteRoles = '' }) { ... }
   ```

3. **Replace app text input with Select dropdown (replaces lines ~161-178):**
   - Remove inline style objects entirely
   - Use `ControlGroup` wrapper with label="App" and required prop
   - Use `Select` with child `Select.Option` elements mapped from `availableApps`:
     ```jsx
     <ControlGroup label="App" required>
     ```

#### Task 2: Multiselect Roles + Password Confirmation with Native Validation (V20, V01, V02)

**Files:** `appserver/static/react/components/CredentialForm.jsx`

**Action:**

1. **Replace readRoles/writeRoles textareas with Multiselect (replaces lines ~200-240):**
   - Change state from comma-separated string to array:
     ```javascript
     const [readRolesArray, setReadRolesArray] = React.useState([]);
     const [writeRolesArray, setWriteRolesArray] = React.useState([]);
     ```
   - Effect on `credential` change: parse comma-separated aclRead/aclWrite into arrays for edit mode. On create mode: initialize from `defaultReadRoles.split(',').map(s => s.trim())` / same for write (to get `['admin', 'power']` defaults per GAP-V03/V04)
   - Use native `<Multiselect>` with chip display and built-in filtering:
     ```jsx
     <ControlGroup label="Read Roles" help="Roles that can view this credential">
       <Multiselect
         values={readRolesArray}
         onChange={(e, { values }) => setReadRolesArray(values)}
         placeholder="Select read roles..."
       >
         {availableRoles.map((role) => (
           <Multiselect.Option key={role} label={role} value={role} />
         ))}
       </Multiselect>
     </ControlGroup>

     <ControlGroup label="Write Roles" help="Roles that can modify this credential">
       <Multiselect
         values={writeRolesArray}
         onChange={(e, { values }) => setWriteRolesArray(values)}
         placeholder="Select write roles..."
       >
         {availableRoles.map((role) => (
           <Multiselect.Option key={role} label={role} value={role} />
         ))}
       </Multiselect>
     </ControlGroup>
     ```
   - **ELIMINATES:** Manual checkbox grid rendering, flex-wrap containers, manual string-to-array parsing in handleSubmit. The `Multiselect` component returns `(string | number | boolean)[]` directly via `onChange` callback's `{ values }` parameter.

2. **Add Confirm Password with native Splunk error validation:**
   - New state: `const [confirmPassword, setConfirmPassword] = React.useState('');`
   - New state for errors (replacing custom red-div approach): `const [errors, setErrors] = React.useState({});`
   - On create mode or edit with `isChangingPassword`, render:
     ```jsx
     <ControlGroup
       label="Confirm Password"
       required={!credential || isChangingPassword}
       error={errors.passwordMismatch}
     >
       <Text
         type="password"
         passwordVisibilityToggle
         value={confirmPassword}
         onChange={(e, { value }) => setConfirmPassword(value)}
         error={!!errors.passwordMismatch}
       />
     </ControlGroup>
     ```

3. **Add pre-submit validation using ControlGroup.error (not custom red div):**
   - Derive mismatch during render (Vercel rule: `rerender-derived-state-no-effect`):
     ```javascript
     // During handleSubmit, set error state:
     if (!credential || isChangingPassword) {
       if (password !== confirmPassword) {
         setErrors({ passwordMismatch: 'Passwords do not match' });
         return;
       }
     }
     ```
   - CLEAR errors on typing in confirm field (Vercel rule: `rerender-move-effect-to-event`):
     ```javascript
     const handleConfirmChange = (e, { value }) => {
       setConfirmPassword(value);
       if (errors.passwordMismatch) {
         setErrors({});
       }
     };
     ```

4. **Update handleSubmit payload - no string parsing needed:**
   - Since Multiselect already returns arrays, serialize directly:
     ```javascript
     onSave({
       username: username.trim(),
       password: password || null,
       realm: realm.trim(),
       app,
       owner,
       readRoles: readRolesArray,
       writeRoles: writeRolesArray,
       sharing,
     });
     ```
   - **ELIMINATED:** `readRoles.split(',').map(r => r.trim()).filter(Boolean)` parsing logic

5. **Clear errors when switching between create/edit modes:**
   - In the credential change useEffect, add `setErrors({})` to clear stale validation state

6. **Ensure password toggle clears confirm field:**
   - When toggling off `isChangingPassword`, also reset confirmPassword and errors

Vercel React Best Practices Applied:
- `rerender-derived-state`: Password mismatch validated at submit time (derived from two state values), not stored in a separate useEffect-computed state
- `rerender-no-inline-components`: Handler functions (`handleConfirmChange`, etc.) defined as named const, not inline in JSX
- `rerender-functional-setstate`: Multiselect onChange uses `{ values }` directly, no need for functional setState since component provides the full new array

**Verify:** Build success. DOM inspection: `<Multiselect>` components present for read/write roles with Splunk-generated chip elements; Confirm Password field visible on both create and edit+changePassword modes; `ControlGroup.error` renders native Splunk error styling (red border, error text below field) on mismatch before submit.

**Done:** Roles rendered as Multiselect dropdowns with chips (read + write sections), confirmPassword uses Text type=password with ControlGroup.error for validation feedback, `sharing` threaded into onSave payload. No custom checkbox rendering, no manual red-div errors, no inline style objects on form controls.
